# Kaggle House Prices Advanced Regression (სახლების ფასების პროგნოზირება)

## პროექტის მიმოხილვა:
    მოცემული პროექტი ეფუძნება Kaggle-ის პლატფორმაზე არსებულ ერთ-ერთ საკვანძო კონკურსს, რომელიც ქალაქ ეიმსში საცხოვრებელი
    სახლების საბოლოო საბაზრო ღირებულების პროგნოზირებას ისახავს მიზნად. ნაშრომი ფარავს მანქანური სწავლების სრულ ციკლს: მონაცემთა 
    კომპლექსურ დამუშავებას, ექსპერიმენტების სისტემატიზებულ მართვას MLflow-ის გამოყენებით და მიღებული შედეგების ვალიდაციას
    Kaggle-ის პლატფორმაზე.

## პრობლემის გადაჭრის მიდგომა:
    პროექტი დავიწყე მონაცემთა წინასწარი დამუშავებით და Baseline მოდელის იმპლემენტაციით, რამაც საშუალება მომცა შემექმნა
    საორიენტაციო წერტილი შემდგომი ეტაპებისთვის. შედარებითი ანალიზისთვის გამოვიყენე რამდენიმე ალგორითმი: Linear Regression,
    Random Forest და XGBoost. მიღებულ მეტრიკებზე დაყრდნობით მოვახდინე საუკეთესო მოდელის იდენტიფიცირება და მისი ინტეგრირება
    DagsHub-ის Model Registry-ში, რაც უზრუნველყოფს მოდელის ვერსიების ეფექტურ მართვასა და ხელმისაწვდომობას.

## რეპოზიტორიის სტრუქტურა:
    model_experiment.ipynb: პროექტის ძირითადი სამუშაო გარემო, რომელიც მოიცავს მონაცემთა სიღრმისეულ ანალიზს, Cleaning პროცესს,
    მოდელების ტრენინგსა და ექსპერიმენტების სისტემატიზაციას MLflow-ის პლატფორმაზე.
    
    model_inference.ipynb: საპროგნოზო გარემო, რომელიც ახდენს რეესტრიდან (Cloud) საუკეთესო მოდელის იმპორტირებას და მის 
    გამოყენებას ტესტური მონაცემების ვალიდაციისთვის.    
    
    submission.csv: მოდელის მიერ გენერირებული საბოლოო ფაილი, რომელიც მზად არის Kaggle-ის პლატფორმაზე ასატვირთად.
    
    train.csv / test.csv: კონკურსის ფარგლებში მოწოდებული საწყისი მონაცემთა სიმრავლეები მოდელის დასამუშავებლად და შესამოწმებლად.

## მონაცემთა დამუშავება (Feature Engineering):
    მონაცემთა ხარისხი პირდაპირ კავშირშია მოდელის ეფექტურობასთან, რის გამოც განვახორციელე შემდეგი ოპერაციები:
    
    Cleaning: მონაცემებში არსებული გამოტოვებული მნიშვნელობები (NaN) ჩავანაცვლე ნულოვანი მნიშვნელობით. ეს მიდგომა ლოგიკურია იმ 
        მახასიათებლებისთვის, სადაც მონაცემის არარსებობა ფიზიკურად ამ ობიექტის არქონას და შესაბამისად, ნულოვან
        გავლენას გულისხმობს.
    
    Feature Selection: ანალიზის ფარგლებში ფოკუსირება მოვახდინე მხოლოდ რიცხვით ცვლადებზე (Numeric Columns), რადგან მათ აქვთ
        ყველაზე მაღალი კორელაცია სამიზნე ცვლადთან (ფასთან).

    Filtering: მონაცემთა სიმრავლიდან ამოვშალე არაინფორმატიული სვეტები (მაგ. Id), რათა მოდელმა თავიდან აიცილოს Overfitting 
        და შეინარჩუნოს განზოგადების უნარი.

## ცვლადების შერჩევა (Feature Selection):
    ფოკუსირება მოხდა მხოლოდ იმ რიცხვით ცვლადებზე (Numeric Columns), რომლებსაც ჰქონდათ ყველაზე მაღალი და სტატისტიკურად
    მნიშვნელოვანი კორელაცია SalePrice ცვლადთან, კატეგორიული ცვლადების ნაცვლად მხოლოდ მაღალკორელირებული რიცხვითი 
    მახასიათებლების გამოყენებამ კი მოგვცა საშუალება, თავიდან აგვეცილებინა Overfitting და შეგვენარჩუნებინა მოდელის მაღალი სიზუსტე 
    ტესტურ მონაცემებზე. 

## მოდელების წვრთნა და ვალიდაცია (Training):
    ექსპერიმენტულ ფაზაში გამოვიკვლიე რამდენიმე ალგორითმი:
    
    Linear & Ridge Regression: გამოყენებულ იქნა, როგორც საბაზისო (Baseline) ორიენტირი შედეგების შესაფასებლად.
    
    Tuned Random Forest: ჩატარდა ჰიპერპარამეტრების ოპტიმიზაცია (n_estimators, max_depth), რამაც გააუმჯობესა მოდელის მიერ 
    რთული კანონზომიერებების აღქმის უნარი.
    
    XGBoost: აღნიშნულმა მოდელმა აჩვენა საუკეთესო ბალანსი სიზუსტესა და გამოთვლით ეფექტურობას შორის.
    
    არჩევანის დასაბუთება: საბოლოო მოდელად განისაზღვრა XGBoost, რადგან მან დააფიქსირა მინიმალური ცდომილება RMSLE მეტრიკის 
    მიხედვით (0.1434), რაც მიუთითებს პროგნოზის მაღალ სიზუსტეზე.


## MLflow Tracking:
    MLflow Tracking: თითოეული ექსპერიმენტის ფარგლებში ხდება ისეთი მეტრიკების მონიტორინგი, როგორიცაა: RMSLE, RMSE, MAE და R^2.
    
    Model Registry: ექსპერიმენტების ფარგლებში შექმნილი ყველა მოდელი ავტომატურად დარეგისტრირდა DagsHub-ის რეესტრში. 
    ეს საშუალებას გვაძლევს, ნებისმიერ დროს გვქონდეს წვდომა მათ სხვადასხვა ვერსიებზე, ხოლო ინფერენსის ეტაპზე
    Cloud-გარემოდან პირდაპირ ჩავტვირთოთ საუკეთესო შედეგის მქონე მოდელი (XGBoost). 

## საბოლოო შედეგი:
    Kaggle Public Score: 0.14057
    Kaggle Public Score (2nd): 0.14315
    საუკეთესო მოდელი: XGBoost

    პროექტის ფარგლებში განხორციელდა 8 სხვადასხვა მანქანური სწავლების ალგორითმის ტესტირება და ვალიდაცია. ამ პროცესში დავაკვირდი
    მოდელების ქცევას და შევარჩიე საუკეთესო: 

    საწყის ეტაპზე გამოყენებულმა წრფივმა მოდელებმა აჩვენეს ყველაზე მაღალი ცდომილება (RMSLE ~0.17). ეს მიუთითებს Underfitting-ზე:
    მარტივი მათემატიკური არქიტექტურა ვერ ახერხებს მონაცემებში არსებული არაწრფივი კანონზომიერებების სრულყოფილად ათვისებას.

    ხეებზე დაფუძნებულმა მოდელებმა მნიშვნელოვნად გააუმჯობესეს პროგნოზირების სიზუსტე (RMSLE ~0.15). დაკვირვებამ აჩვენა, რომ 
    ბაზისურ და ოპტიმიზებულ (Tuned) Random Forest-ს შორის მცირე სხვაობა იყო, რაც მიანიშნებდა, რომ მოდელმა მიაღწია თავისი 
    შესაძლებლობების მაქსიმუმს მოცემულ მონაცემებზე.

    საუკეთესო შედეგი დაფიქსირდა XGBoost-ის გამოყენებისას (RMSLE 0.14). ჰიპერპარამეტრების ფრთხილმა შერჩევამ (მაგ. learning_rate,
    max_depth) მოგვცა საშუალება, თავიდან აგვეცილებინა ბუსტინგისთვის დამახასიათებელი Overfitting-ის რისკი და მიგვეღო ბალანსი
    მოდელის სირთულესა და სიზუსტეს შორის.
